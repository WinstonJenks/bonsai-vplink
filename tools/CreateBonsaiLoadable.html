NAME

CreateBonsaiLoadable - Creates a VP Link loadable file for use with Bonsai

SYNOPSIS

CreateBonsaiLoadable.exe usage: CreateBonsaiLoadable.exe [-h] [--dir TARGETDIR] [--name NAME] [--list] [--progress] [--nowork] [--version]

DESCRIPTION

CreateBonsaiLoadable is a program that will create a sim.zip file for use with Microsoft Bonsai. This sim.zip file is called a "VP Link loadable".

The user needs to specify the root of the loadable--this is the directory containing a vplink_interface.json file as well as other files under the ./cfg/ directory.

OPTIONS

--dir TARGETDIR

TARGETDIR is a directory which is the root of the VP Link loadable. It should contain a vplink_interface.json file created using the CreateBonsaiInterface.exe program. The default value is the current directory (./).

--name NAME

This is the name of the created file. The default is sim.zip. By convention this is typically named <MySimName>_sim.zip to keep various loadable files organized on your local file system. Note that you can drop a file with any name (as long as it is a .zip) on the Bonsai web site to create your VP Link based simulator.

--list

This option will list the contents of the loadable specified by the --dir and --name options. Use this to verify the contents of the loadable file.

--progress

Instructs the program to emit additional progress information to stderr. This can be specified multiple times to increase the verbosity of the output.

--nowork

Do almost everything, but do not create an output file. Useful to use as a dry run.

--version

Print the version of this program and exit.

--help

Prints the usage summary. Looks like this:

usage: CreateBonsaiLoadable.exe [-h] [--dir TARGETDIR] [--name NAME] [--list] [--progress] [--nowork] [--version]

optional arguments: -h, --help show this help message and exit --dir TARGETDIR specify the directory to use as the root of the loadable --name NAME the name of the loadable --list lists contents of the named loadable --progress print additional information while running --nowork do not do anything, just pretend --version, -v prints the version

EXAMPLE

CreateBonsaiLoadable.exe --dir loadable\QT5 # Creates a sim.zip file in the loadable\QT5\ directory
